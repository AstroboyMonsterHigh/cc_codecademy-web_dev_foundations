.activity { }
The base style. Applies always as the foundation — other states build on top of this.
Always active
L
.activity:link
Targets an <a href> that has never been visited. Fresh links the user hasn't clicked yet.
Has href + unvisited
V
.activity:visited
Fires when the browser recognizes the URL from history. Only color/background can be changed (privacy rules).
URL is in history
H
.activity:hover
Active while the mouse cursor is over the element. Works on any element — divs, buttons, images, links.
Mouse is over element
A
.activity:active
Fires during the split second of clicking — from mousedown to mouseup. Also fires on keyboard press.
Being clicked right now
⚡ Correct order: :link → :visited → :hover → :active

❌ Wrong order HOVER BROKEN ON VISITED

:hover is declared before :link and :visited. Since visited/link come after in the stylesheet, they override hover — so hovering a visited link shows no hover effect!

google.com (try hovering!) unvisited link (hover works)
/* ❌ WRONG ORDER */ .link:hover { color: orange; } ← 1st .link:link { color: blue; } ← 2nd (overrides hover!) .link:visited { color: purple; } ← 3rd (overrides hover!) .link:active { color: red; } ← 4th Result: hover style is LOST on visited or unvisited links because :link and :visited declared after :hover win the cascade!
✅ LVHA Order — Correct!

:link and :visited set the base state. :hover comes after, so it can override both. :active comes last to win over everything while clicking.

google.com (hover works!) unvisited link (hover works)
/* ✅ LVHA — CORRECT ORDER */ .link:link { color: blue; } ← 1st .link:visited { color: purple; } ← 2nd .link:hover { color: orange; } ← 3rd (overrides L+V ✓) .link:active { color: red; } ← 4th (overrides H ✓) Result: hover ALWAYS works whether the link is visited or not. Active always fires on click. Every state wins at the right moment!

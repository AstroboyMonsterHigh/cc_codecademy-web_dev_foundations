What is LVHA?

L Lord / Link a:link Unvisited links
V Vader / Visited a:visited Already clicked
H Hates / Hover a:hover Mouse over it
A Anakin / Active a:active Click & hold
Why does order matter? CSS reads rules top-to-bottom. If :hover comes before :visited, a visited link being hovered won't show the hover color — the :visited rule below will override it (they have equal specificity). The LVHA order ensures each state can be overridden naturally by later states. Think of it as a cascade: L → V → H → A each "wins" over the previous.

Order matters — see the bug!

Watch out
❌ Wrong order

After visiting, hover won't turn red — :visited overrides :hover

a:link { color: #b48eff } a:hover { color: red } ← 2nd a:visited { color: gray } ← 3rd overrides! a:active { color: lime }
✅ Correct order
Click me first, then hover Hover on a visited link

Even after visiting, hover still glows orange!

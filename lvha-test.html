📖 LVHA order explained

a:link { } → a:visited { } → a:hover { } → a:active { }
✨ Why? Because hover & active should override link & visited styles. If you swap, weird things happen! Love = LVHA (LoVe / HAte mnemonic). Try to reorder in devtools — you'll see ❤️

✅ In the right column, all four states work in harmony. Hover over any link → warm orange; click → active scale effect; visited links show soft mauve background!

🎀 What each state means

  • :link — an unvisited link (has href, not visited yet).
  • :visited — a link the user has already clicked (browser remembers!).
  • :hover — mouse cursor over the element.
  • :active — moment when link is being clicked (mouse down).

🧁 Common mistakes

  • ❌ Writing :hover before :link breaks spec – hover won't override unvisited style properly.
  • ❌ Forgetting :visited styles → visited links look same as unvisited, confusing UX.
  • ✅ Always stick to LVHA order for consistent, accessible styling!
